# BrainSim 🧠

**BrainSim** — это интерактивный симулятор спайковой нейронной сети (Spiking Neural Network, SNN), написанный полностью на Swift без использования внешних ML-библиотек.

В отличие от классических нейросетей (как ChatGPT или обычные классификаторы), которые оперируют числами с плавающей точкой (векторами) и обучаются через Backpropagation, BrainSim эмулирует работу мозга на уровне отдельных биологических нейронов и их спайков во времени. Обучение происходит локально и непрерывно за счет биологически правдоподобных правил (STDP).

## Зачем нужен этот проект?

Проект создан как **наглядный прототип и песочница** для изучения и демонстрации принципов работы третьего поколения нейросетей (SNN). 

**Для чего он пригоден:**
- **Изучение биологического обучения:** Понять, как локальное правило STDP (Spike-Timing-Dependent Plasticity) способно заставить сеть находить и запоминать закономерности без учителя.
- **Визуализация динамики:** Увидеть "живое" поведение сети в реальном времени (синхронизация спайков, волны активности).
- **Эксперименты с параметрами:** Тестировать влияние различных типов нейронов (регулярно спайкующие, быстро спайкующие) и параметров STDP на стабильность сети.
- **Основа для портирования:** Логика вычислений (Core) спроектирована так, чтобы её можно было перенести 1:1 на более производительные языки (например, Rust) для масштабирования на миллионы нейронов. Swift идеален для UI и визуализации.

**Для чего он НЕ пригоден:**
- Это не production-ready фреймворк для решения прикладных ML задач (распознавание котиков, LLM).
- Симуляция ограничена сотнями нейронов из-за накладных расходов SwiftUI и Canvas на каждый кадр. Для масштабных симуляций требуется перенос `Core` на GPU/Metal или Rust.

## Архитектура и технологии

Проект построен на нескольких ключевых биологических концепциях:

1. **Модель нейрона Ижикевича (Izhikevich, 2003):** 
   Вместо тяжелых моделей Ходжкина-Хаксли, здесь используется система из двух дифференциальных уравнений. Она вычислительно легкая, но способна воспроизводить десятки реальных паттернов поведения нейронов (Regular Spiking, Fast Spiking Inhibitory и др.).
2. **STDP (Spike-Timing-Dependent Plasticity):**
   Алгоритм обучения. Если нейрон А спайкует прямо перед нейроном Б, связь (синапс) между ними усиливается. Если наоборот — ослабляется. Это позволяет сети самостоятельно находить причинно-следственные связи во входных данных.
3. **Гомеостаз (Homeostasis):**
   Встроенный механизм регуляции, который не дает сети "взорваться" от эпилептической активности или полностью затухнуть. Нейроны динамически масштабируют свою чувствительность, чтобы поддерживать целевую частоту спайков.

## Что можно тестировать и демонстрировать?

Запустите проект в Xcode (macOS таргет) и используйте встроенные вкладки для экспериментов.

### 1. Формирование кластеров (Raster Tab)
- **Что делать:** Переключите входной сигнал (Input Mode) с `Random` на `Pattern A`. Оставьте скорость (Speed) на комфортном уровне.
- **Что наблюдать:** На графике "Spike Raster" (время по X, нейроны по Y) вы увидите, как изначально хаотичные спайки начинают группироваться в четкие вертикальные паттерны. Это означает, что STDP усилил связи между нейронами, которые активируются вместе, заставляя их стрелять синхронно.

### 2. Эксперимент на специализацию (Experiment Tab)
- **Что делает:** Демонстрирует, что сеть действительно "обучается" различать разные входы без учителя.
- **Как запустить:** Перейдите на вкладку `Experiment`. Выберите количество шагов тренировки (Train steps) и нажмите `Run Experiment`.
- **Процесс:** Сеть будет принудительно кормиться паттерном `Pattern A` в течение N шагов (обучение). Затем она будет протестирована короткими вспышками `Pattern A` и `Pattern B` с отключенным обучением.
- **Результат:** Вы увидите гистограмму селективности (Selectivity). Зеленые столбики — нейроны, которые научились реагировать *только* на паттерн A (score > 0.5). Красные — нейроны, которые реагируют *только* на незнакомый паттерн B. Это показывает, что сеть выделила группы нейронов-детекторов под конкретные стимулы.

### 3. Реакция на тишину (Homeostasis check)
- **Что делать:** После обучения сети переключите Input Mode в `Silence`.
- **Что наблюдать:** Сеть не замолкнет мгновенно. Некоторое время по ней будут "гулять" эхо-волны спайков по уже усиленным путям. Затем гомеостаз начнет задирать чувствительность нейронов из-за отсутствия входа, что может привести к спонтанным случайным спайкам из шума.

## Структура файлов

- `Core/Neuron.swift`: Математика нейрона Ижикевича.
- `Core/Synapse.swift`: Связь со встроенной логикой изменения весов (STDP).
- `Core/Network.swift`: Контейнер сети, управляющий обходом всех нейронов за один такт (`step`).
- `Runtime/Simulator.swift`: Движок симуляции, работающий в фоновом потоке и публикующий данные для UI.
- `Runtime/PatternExperiment.swift`: Изолированный скрипт проведения тестирования на специализацию.
- `UI/`: Набор SwiftUI Canvas компонентов для отрисовки растра, гистограмм и панелей управления с высокой частотой кадров.
